# 전자혀 (Electronic Tongue) — 요똥을 위한 맛 튜닝 앱

요리 초보(요똥)가 **음식 무게**와 **원하는 맛 스타일**을 고르면, 조미료 배합을 **g 단위 + 숟가락 환산**으로 계산해주는 단일 파일 웹앱입니다. 빌드·설치·로그인·서버 없이 `index.html` 하나로 동작합니다.

## 실행

```bash
# 가장 간단 — 파일을 더블클릭
index.html

# 또는 로컬 서버 (폰에서 같은 와이파이로 접속하려면)
python -m http.server 8000
# → PC:  http://localhost:8000
# → 폰:  http://<PC의-LAN-IP>:8000   (예: http://192.168.0.10:8000)
```

> 모바일 우선 설계입니다. 폰 브라우저에서 화면을 꽉 채우고(노치/홈바 안전영역 반영), 슬라이더·버튼은 터치 타깃 44px 이상으로 키웠습니다.

## 기능 (하단 탭 4개)

| 탭 | 내용 |
|---|---|
| 🧮 **계산** | 카테고리·무게·스타일·강도 → 양념 배합표(g + 숟가락 + 넣는 타이밍) · 맛 프로필 6축 · 조리 순서 · 안전 가이드 · 저장 |
| 🩹 **보정** | 망한 음식 증상 10종 → 1차/2차 처방 + 그램 환산. 짠 음식엔 소금 추가 금지 강제 |
| 🎚️ **탐색** | 5축 슬라이더로 "존맛 지수"를 실시간 예측하는 배합 시뮬레이터(셰프 미강) + 역최적화 프리셋 |
| ★ **저장** | 계산 결과를 localStorage에 보관 · 별점 · 재료 보기 · 삭제 |

## 맛 계산 엔진

음식의 맛을 6축(짠·단·신·기름·감칠·향/매운)으로 해석한 **요리 실무 기준 상대값 휴리스틱**입니다.

- **템플릿 기반**: 검증된 스타일(아이용 간장버터·핫허니 양념치킨·이연복식 중식)은 300g 기준 원형 레시피를 무게에 맞춰 스케일링
- **목표값 기반**: 그 외 스타일은 `음식무게 × 카테고리 목표% × 스타일 보정`으로 맛 등가량을 구한 뒤, 추천 조미료에 "주재료 70% + 보조재료 30%"로 배분
- 단위 환산: 재료별 밀도로 `ml = g / density` → 작은술(5ml)·큰술(15ml)
- 강도(순한 0.85 / 기본 1.0 / 진한 1.15), 단 산미·매운맛은 1.15에서도 1.1배까지만

조미료 25종+, 음식 카테고리 11종, 맛 스타일 13종, 보정 규칙 10종 내장. 자세한 사양은 [docs/PRD.md](docs/PRD.md) 참고.

## 검증 (수동 테스트 시나리오 — PRD §19)

| 테스트 | 기대 | 결과 |
|---|---|---|
| 아이용 닭볶음 300g | 간장 12~18g, 올리고당 6~12g, 버터 8~15g, 산미·매운 없음 | ✅ 15 / 10 / 10g |
| 로제떡볶이 500g | 고추장·케첩·우유·치즈·버터 조합, 지방감 높음 | ✅ |
| 중식 볶음 300g | 굴소스·간장·설탕·기름·전분물·마늘·생강, 기름 8~15% | ✅ 8.3%, 전분물 마무리 |
| 서양식 구이 300g | 소금·버터·올리브오일·레몬·마늘, 단맛 ~0, 마지막 산미 | ✅ |
| 보정: "짜기만 해요" | 물 희석 + 소금 추가 금지 | ✅ |

## 주의

계산값은 제품별 조미료 농도 차이와 개인 취향에 따라 달라질 수 있습니다. **짠맛·신맛·매운맛은 처음에 80%만 넣고 맛보며 조절하세요.**
